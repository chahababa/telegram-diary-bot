# 📔 Telegram 日記 Bot — 專案進度紀錄

## 2026-03-29｜第一次開發（從零開始）

### 今天做了什麼

今天從零開始做了一個 Telegram 日記助理 Bot。這個 Bot 的用途是：我每天隨時傳文字或語音給它，它會幫我記下來，晚上自動整理成一篇完整的日記，還會上傳到 Google Drive。

整個過程是跟 AI（Claude）一步一步合作完成的，因為我自己不太會寫程式，所以每一步都有解釋。

### 完成的事情

1. **建好整個專案架構**：用 Python 寫的，總共 17 個檔案，大約 1,600 行程式碼
2. **核心功能都做好了**：
   - 傳文字訊息 → 自動存起來
   - 傳語音訊息 → AI（Whisper）幫忙轉成文字再存起來
   - 每天 5 個時段會提醒我記日記（09:00、12:00、15:00、18:00、21:00）
   - 晚上 23:00 Bot 會問我幾個問題（今天最重要的事、感恩三件事、心情評分）
   - 凌晨 00:00 AI（GPT-4o）自動把一整天的記錄整理成 Markdown 日記
   - 日記會上傳到 Google Drive（還沒設定憑證，目前先存本地）
3. **管理員指令**：我可以在 Telegram 裡面直接改設定，不用動程式碼。例如改提醒時間、改日記範本、改提醒訊息
4. **部署到雲端**：用 Zeabur 部署（東京伺服器），連結 GitHub，推 code 上去就會自動更新
5. **寫好專案文件**：README.md（專案總覽）和 PROJECT_STATUS.md（詳細狀態說明書）

### 遇到的問題

- 語音辨識連續失敗 3 次 → 原因是 OpenAI 的客戶端用錯版本（同步 vs 非同步），已經修好了但還沒測試
- Git 初始化一開始出了一些小問題，後來解決了
- Zeabur 部署第一次也 crash 了，因為忘記加環境變數

### 還沒做完的事

- **語音辨識修復後要測試**：程式碼已經改好，但還沒推上 GitHub 讓 Zeabur 重新部署。需要在終端機跑 `git add -A` → `git commit` → `git push`，然後到 Telegram 測試語音
- **Google Drive 上傳還沒設定**：需要去 Google Cloud 建立 Service Account，拿到 credentials.json
- **管理員改設定後要重啟才生效**：目前改完提醒時間要到 Zeabur 手動 Restart，未來可以改成自動生效

### 未來想做的

- 做一個網頁後台可以瀏覽歷史日記
- 心情趨勢用圖表呈現（目前只有文字）
- 整合 4 間好初早餐的營運記錄
